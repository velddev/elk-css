intro

Hey there bud.
Tired of fighting with other frameworks? Tired of CSS's overall of buggyness? Want something that just works™?

Well then - elk-css may be for you!
elk tries to be a simple as possible, its a framework that skips any major UI changes, but gives you enough of a jump start to get you going with a responsive base, so you can create great looking sites in no time.

Standard 12 column grid

Here's your standard 12 column grid. Nothing new, does exactly what you'd expect.

The container has a max-width of 1200px for those with big ole' tellies with a standard width of 80vw.

The whole thing has a single breakpoint, one for mobile & one for desktop, which breaks at 650px, go ahead - resize your browser!

syntax

Essentially it comes down to 3 components.

  • Define the type of container (row or column)
  • Specify its width / pull / margins + paddings
  • Specify what should happen internally

And the output:

Center the contents

Right align the contents

classes

.container

Contains everything, 80vw.

.row

One line, everything goes inline

.column

Can span multiple lines, everything flows downwards

.half

Fill half of parent

.third

Fill one third of parent

.two-thirds

Fill two-thirds of parent

.full

Fill entire row/column line (sometimes not needed)

.stretch

Fill entire container, useful for nested columns divs

.vcenter / .hcenter

Horizontally / vertically center a div

.pull-left / .pull-right

Pull a div to the left / right of parent

.circular

Make the div into a circle / oval

.vspacer .hspacer

Kinda like <br> but vertically and horizontally

.tal / .tac / .tar

taX = text-align-left/center/right

.nm / .np

Makes div have no padding/margin

.nav

Useful for navs

.relative

Parent class for containing divs with internal class

.internal

Child class for .relative, see hints for more info

.left / .right / .bottom / .top

Internal suffixes that specific where to place contents

.col-2/4/6/8/10/12

Column width, 2, 4, 6, 8, 10, 12

.bl

Force to align-items to baseline instead of flex-start


So you can make some otherwise complex layouts pretty easily.

typography

elk-css likes big fonts.

Default fonts are; Nunito Sans, for most text and Inconsolata for code, both of which are served by Google.
All headers are specified in vw sizes, so that they scale to screen width.

Everything else has a size of 1rem.
